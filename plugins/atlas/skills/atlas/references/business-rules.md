# Big Uppetite — Business Rules

These rules are non-negotiable. They reflect how Big Uppetite operates and must be applied in every ATLAS output.

## 1. Clients Build, Team Supports

Cards should reflect client responsibility where applicable. Use language like:
- ✅ "Teach [Client] to set up their email sequences"
- ❌ "Build email sequences for [Client]"

Exception: if the task was explicitly sold as a done-for-you service, write it accordingly.

## 2. 12-Week Programme

Every client is on a 12-week engagement timeline. Track `programme_week` from the registry. During each health check, assess whether visible progress matches the client's current week. Flag any client where progress appears behind.

## 3. Pause Policy

Clients may pause a maximum of **twice**, up to **2 weeks** each pause. Track pauses in `client_registry.md`. If a client has been inactive for 14+ days and no pause is logged, flag it as potentially at risk.

## 4. No Scope Creep

If a new task arises that wasn't in the original service agreement, flag it before creating the card. Never silently absorb scope. Always surface it to Jenna for a decision.

## 5. Money Loves Speed

When multiple implementation options exist, default to the **simplest, fastest version** unless instructed otherwise. Flag when the user is choosing complexity over speed.

## 6. Communication Language

All client-facing content must be:
- Positive and action-oriented
- Never tell a client what can't be done
- Frame everything as what will be done and by when
- Use future tense: "We'll have this ready by..." not "We can't do this until..."

## 7. Evidence of Work

Every completed task should leave a trail. Encourage:
- Trello comments on completed cards
- Completion notes with timestamps
- Screenshots or deliverable links attached to cards

When writing a card that involves a deliverable, add a "COMPLETION NOTE" placeholder at the bottom of the description so the assignee knows to document their work.
